Split pools by job type

uv run prefect work-pool create extract --type process   # I/O — generous --limit
uv run prefect work-pool create encode  --type process   # CPU — GCL-capped (L11)

# Each machine runs TWO workers, one per pool, each with its own limit:
WORKER_ID=big-1 uv run prefect worker start --pool extract --limit 12
WORKER_ID=big-1 uv run prefect worker start --pool encode  --limit 4

Why separate, not one pool Head-of-line blocking. In one shared pool a backlog of slow, tightly-capped encodes parks every worker slot and quick unpacks wait behind them. Separate pools = independent limits, scaling, and routing. (Two queues in one pool is the lighter alternative; separate pools is cleaner and lets a GPU box be its own pool.)

Fan out: one run per unit of work

from prefect import flow
from prefect.deployments import run_deployment

@flow   # deployed to the `extract` pool
def process_archive(archive: str):
    segments = unpack(archive)            # light I/O, runs here
    runs = [
        run_deployment(
            "encode-segment/encode",      # "flow-name/deployment-name"
            parameters={"segment": s},
            timeout=0,                    # fire-and-forget: return at once
        )
        for s in segments
    ]
    return [r.id for r in runs]           # one DISTRIBUTED flow run per segment

run_deployment knobs

ArgEffect
timeout=0Return immediately after the run is created (fan-out). Default (None) blocks until it finishes; a number waits up to N seconds.
parameters={…}Per-run overrides, merged with the deployment defaults.
as_subflow=TrueDefault. Nests the run under the caller in the UI — does not co-locate execution; it still runs on the pool.
idempotency_key=…Dedupe: a retried coordinator won't create duplicate encode runs for the same key.
work_queue_name=…Target a specific queue within the deployment's pool (e.g. route by machine class).

Gather the fan-out L13

from prefect.flow_runs import wait_for_flow_run

runs = [run_deployment("encode-segment/encode",
                       parameters={"segment": s}, timeout=0,
                       idempotency_key=f"{archive}:{s}")   # stable → safe coordinator retry
        for s in segments]                                 # fire ALL first

finished = [wait_for_flow_run(r.id) for r in runs]         # then gather each (async-only ⚠)

done, failed = [], []
for fr in finished:
    if fr.state.is_completed():
        done.append(fr.state.result())                     # needs persisted results (L4)
    else:
        failed.append(fr.parameters["segment"])            # raise_on_failure=False → record, don't sink
NeedUse
All running at once, results laterFire every run with timeout=0, then wait_for_flow_run(id) on each — never block as you create.
Did each finish?fr.state.is_completed() / is_failed() — state lives in the server DB, always available.
The return value across machinesfr.state.result() — needs persisted results in shared storage (MinIO/S3, L4); otherwise gather states only.
Keep good results when some failstate.result(raise_on_failure=False) returns the exception instead of raising. Filter, re-dispatch only failures.
Safe coordinator retryidempotency_key from stable inputs (archive:segment) — a rerun reuses finished runs, never double-encodes. Random keys defeat it.

Async coordinator L14

import asyncio
from prefect.client.orchestration import get_client
from prefect.deployments import arun_deployment
from prefect.flow_runs import wait_for_flow_run

@flow
async def process_archive(archive: str):
    runs = await asyncio.gather(*(                          # phase 1: fire all, concurrently
        arun_deployment("encode-segment/encode",
                        parameters={"segment": s}, timeout=0,
                        idempotency_key=f"{archive}:{s}")
        for s in segments))

    finished = await asyncio.gather(*(                       # phase 2: gather, concurrently
        wait_for_flow_run(r.id) for r in runs))              # the poll loop is GONE

    async with get_client() as client:                       # ⚠ settle the race (see gotchas)
        finished = await asyncio.gather(*(client.read_flow_run(fr.id) for fr in finished))

    for fr in finished:
        res = await fr.state.aresult(raise_on_failure=False) # async result: aresult, NOT result.aio
Naive sync (breaks here)Async (L14)
run_deployment(...)await arun_deployment(...) (or run_deployment.aio(...))
hand-rolled get_client(sync_client=True) poll loopawait wait_for_flow_run(id) — the documented gather API
fr.state.result(...)await fr.state.aresult(...) (.result.aio drops self ⚠)
list comprehension (serial creates)asyncio.gather — concurrent dispatch (matters at high fan-out)
What async does / doesn't buy The encodes already run in parallel — that's timeout=0 + the GCL, in both versions. Async buys: the documented wait_for_flow_run API, concurrent dispatch (N creates at once, not back-to-back — invisible at 6 segments, the whole cost at 600), and deleting the poll loop. Not a wall-clock win on small fan-outs.

Gotchas

Things that bite
  • wait_for_flow_run is async-only (verified, 3.7.5) → in a sync coordinator it returns a coroutine. Make the flow async and await wait_for_flow_run (L14); the sync fallback is the get_client(sync_client=True).read_flow_run(id) poll loop shown above.
  • wait_for_flow_run hands back a racing state (L14) → it's event-driven (poll_interval ignored): returns on the first final-state event, then re-reads, and that re-read can lag the commit — a finished run can come back RUNNING (seen ~1 run in 5), so it looks "failed". Re-read the run's state before classifying; one read settles it.
  • fr.state.result.aio(...) → the bound-method .aio drops self in 3.7.5. Use await fr.state.aresult(...).
  • Using .map for fleet scale → it stays in one process on one machine. Use run_deployment per unit.
  • Expecting as_subflow to co-locate → it's lineage only; execution is on the pool.
  • Blocking the coordinator → forgetting timeout=0 makes the parent wait on each encode serially. Fire all, gather later.
  • Wrong deployment name → it's "flow-name/deployment-name", not the file or function name.
