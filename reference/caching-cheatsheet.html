Caching & cache policies

Skip work that already succeeded — every policy and knob in one place, copy-paste ready.

The decorator arguments

ArgumentWhat it does
cache_policy=POLICYHow the cache key is computed. A hit returns the stored result as a Cached state — your function never runs.
cache_expiration=timedelta(...)How long a cached result stays valid. After it lapses, the next call recomputes. Omit = never expires.
cache_key_fn=fnA callable (context, parameters) -> str for a fully custom key. Use when the built-in policies don't fit.
persist_result=TrueStore results to disk so they survive across runs. Setting a cache_policy turns this on for you.

Built-in cache policies

Import from prefect.cache_policies. Combine them with +.

PolicyKeys on
DEFAULTInputs + task code + run id. The catch: RUN_ID means it only hits within one flow run.
INPUTSThe task's arguments only. Hits across runs — this is usually what you want for scheduled pipelines.
TASK_SOURCEThe task's source code lines. Editing the task busts the cache.
FLOW_PARAMETERSThe parameters passed to the parent flow.
NO_CACHEDisables caching entirely.

Cache on inputs, with expiration

from datetime import timedelta
from prefect import task
from prefect.cache_policies import INPUTS

@task(cache_policy=INPUTS, cache_expiration=timedelta(minutes=5))
def expensive_fetch(source):
    ...  # same `source` within 5 min -> Cached, no re-run

Re-run when inputs OR code change

from prefect import task
from prefect.cache_policies import INPUTS, TASK_SOURCE

@task(cache_policy=INPUTS + TASK_SOURCE)
def transform(rows):
    ...  # busts the cache if you edit the body

The composite default, spelled out

DEFAULT = INPUTS + TASK_SOURCE + RUN_ID

Why your cache "doesn't work" across runs The default includes RUN_ID, so a fresh run never reuses a previous run's results. To cache between scheduled runs, pick a policy without RUN_ID — e.g. INPUTS or INPUTS + TASK_SOURCE.

Mental model

WantRight tool
Skip a slow call when args are unchangedcache_policy=INPUTS
Also re-run when you edit the codecache_policy=INPUTS + TASK_SOURCE
Treat data as stale after a whilecache_expiration=timedelta(...)
Cache only inside a single run (safe default)DEFAULT (the built-in behavior)
Turn caching off for one taskcache_policy=NO_CACHE
Caching vs retries Different jobs. Retries re-run work that failed; caching avoids re-running work that succeeded. They compose: a task can retry on a blip and, once it lands, persist a cached result for next time.
