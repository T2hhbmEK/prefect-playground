Diagnose a failed run

Read run history after the fact — which task broke, why, and when. UI + CLI, copy-paste ready.

First: which state?

StateMeansLook in
FailedYour code raised an exceptionYour code / the data — read the traceback
CrashedInfra died (OOM, killed, power)The machine / process, not your code
CompletedSucceeded—

Why state first Failed vs Crashed decides where you even look. A traceback exists for Failed; a Crashed run may have none, because the process vanished before it could report.

The CLI workflow

# 1. Find the failure
uv run prefect flow-run ls --state Failed --limit 5
uv run prefect flow-run ls --flow-name nightly-report      # or by flow

# 2. Inspect state + failure message
uv run prefect flow-run inspect <flow-run-id>

# 3. Read the full logs / traceback
uv run prefect flow-run logs <flow-run-id>

# 4. Re-run it once you've fixed the cause
uv run prefect flow-run retry <flow-run-id>

prefect flow-run commands

CommandUse
lsList recent runs. Filter: --state Failed, --state-type FAILED, --flow-name <name>, --limit N.
inspect <id>Full record of one run: state, message, timing, parameters.
logs <id>All log lines for the run — the traceback lives here.
retry <id>Re-run a failed (or completed) run.
watch <id>Follow a run live until it reaches a terminal state.
cancel <id> / delete <id>Stop a run / remove its record.

In the UI (localhost:4200)

WhereWhat it tells you
Flow Runs, filter FailedEvery failed run, newest first.
A run's task-run timelineWhich task failed and how far the pipeline got (Completed tasks before it).
The failed task's Logs tabThe exception and traceback for that task.
The run's Parameters / inputsThe exact inputs that triggered the failure — reproduce it.
