Failure alerts

Get told when a run fails — hooks for per-flow logic, Automations for fleet-wide safety nets.

Two mechanisms — pick by need

State-change hookAutomation
Runs whereIn your flow's processOn the Prefect server
Catches Failed✓✓
Catches Crashed (process killed)✗ (no process left to run it)✓ (server sees the event)
ScopeOne flow (custom per-flow logic)Every deployment, one rule
Defined inPython (@flow(on_failure=[...]))UI or Python SDK

on_failure hook

from prefect import flow

def alert_on_failure(flow, flow_run, state):
    msg = f"Flow '{flow_run.name}' entered {state.name}.\nReason: {state.message}"
    SlackWebhook.load("ops-alerts").notify(body=msg, subject="🔴 a run failed")

@flow(on_failure=[alert_on_failure])
def nightly_report():
    ...

Other flow hooks: on_completion, on_crashed, on_cancellation, on_running. All receive (flow, flow_run, state).

Notification blocks

A reusable connection to a channel. Save once, .load() + .notify(body, subject) anywhere.

# Production — real Slack
from prefect_slack import SlackWebhook
SlackWebhook(url="https://hooks.slack.com/services/...").save("ops-alerts")

# Local learning — POST to a catcher (alert_listener.py)
from prefect.blocks.notifications import CustomWebhookNotificationBlock
CustomWebhookNotificationBlock(
    name="local-alert",
    url="http://127.0.0.1:8077/alert",
    method="POST",
    json_data={"text": "{{subject}}\n{{body}}"},
    headers={"Content-Type": "application/json"},
    allow_private_urls=True,   # required for localhost
).save("local-alert", overwrite=True)

Built-in blocks (in prefect.blocks.notifications): SlackWebhook, MicrosoftTeamsWebhook, DiscordWebhook, MattermostWebhook, OpsgenieWebhook, CustomWebhookNotificationBlock, plus Apprise-backed channels.

Server-side Automation (SDK)

from prefect.automations import Automation
from prefect.events.schemas.automations import EventTrigger, Posture
from prefect.events.actions import SendNotification  # or configure in the UI

Automation(
    name="alert-on-failure",
    trigger=EventTrigger(
        expect={"prefect.flow-run.Failed", "prefect.flow-run.Crashed"},
        match={"prefect.resource.id": ["prefect.flow-run.*"]},
        posture=Posture.Reactive,
        threshold=1,
    ),
    actions=[...],   # send notification, etc.
).create()

Easiest path Build it in the UI: Automations → New → trigger "Flow run state" (Failed/Crashed) → action "Send a notification" (pick a block). One rule, every deployment.
