Results & object storage

Caching big artifacts to S3-compatible object storage (incl. Tencent COS) — every knob in one place.

The golden rule

Cache the pointer, store the payload Prefect caches a task's return value. For a big artifact (mp4, model, Parquet), upload it yourself and return its key — never the bytes. result_storage holds the return value; your own upload puts the file in COS. Two separate jobs.

The decorator arguments

ArgumentWhat it does
result_storage=<block>Where Prefect writes the (small) return value. A storage block, or its slug e.g. "s3-bucket/cos-video-bucket". Inherited by tasks from the flow if unset.
persist_result=TrueForce results to disk/object store so they survive the process. Setting a cache_policy turns this on for you.
result_storage_key="..."Deterministic name for the stored result (templatable).
cache_policy=INPUTSKey the cache on the task's args. Same args across runs ⇒ Cached. (See caching cheatsheet.)

The artifact pattern

import hashlib
from pathlib import Path
from prefect import task
from prefect.cache_policies import INPUTS
from prefect_aws import S3Bucket

cos = S3Bucket.load("cos-video-bucket")   # saved earlier

def content_hash(path: Path) -> str:
    return hashlib.sha256(path.read_bytes()).hexdigest()[:12]

@task(cache_policy=INPUTS, result_storage=cos)
def encode_video(src_hash: str, preset: str = "h264-1080p") -> str:
    out_key = f"encoded/{src_hash}/{preset}.mp4"
    local_mp4 = run_ffmpeg(...)                 # transcode -> local file
    cos.upload_from_path(local_mp4, out_key)    # the artifact upload
    return out_key                              # the small string Prefect caches

Key on content, not name Pass a content hash + preset to the task, not a file path. A path-keyed cache returns a stale encode when the bytes change and a wrong one when the preset changes.

Local MinIO block (for learning locally)

MinIO is an S3-compatible store you run yourself: docker compose up -d (console at :9001). Use MinIOCredentials.

from prefect_aws import MinIOCredentials, AwsClientParameters, S3Bucket

cos = S3Bucket(
    bucket_name="prefect-artifacts",
    credentials=MinIOCredentials(
        minio_root_user="minioadmin",
        minio_root_password="minioadmin",
        aws_client_parameters=AwsClientParameters(endpoint_url="http://localhost:9000"),
    ),
)
cos.save("minio-artifacts", overwrite=True)

MinIO ⇄ COS is a credentials swap Both are S3-compatible, so the task code (upload_from_path, result_storage=cos) is identical. Only the block changes: MinIOCredentials + localhost:9000 ⟶ AwsCredentials + the COS endpoint below.

Tencent COS block (verified vs. prefect-aws)

from prefect_aws import AwsCredentials, AwsClientParameters, S3Bucket

cos = S3Bucket(
    bucket_name="videos-1250000000",          # name MUST include your APPID
    credentials=AwsCredentials(
        aws_access_key_id="<SecretId>",        # Tencent CAM SecretId
        aws_secret_access_key="<SecretKey>",   # Tencent CAM SecretKey
        region_name="ap-guangzhou",
        aws_client_parameters=AwsClientParameters(
            endpoint_url="https://cos.ap-guangzhou.myqcloud.com",
        ),
    ),
)
cos.save("cos-video-bucket", overwrite=True)

SecretId / SecretKeyaws_access_key_id / aws_secret_access_key

S3Bucket I/O methods

MethodUse
upload_from_path(local, key)Upload a local file to the bucket.
download_object_to_path(key, local)Download an object to a local file.
read_path(key) / write_path(key, bytes)Read/write raw bytes by key. read_path doubles as an existence check.
aupload_from_path, aread_path, …a-prefixed = async variants.

The stale-pointer gotcha

Cache hit ≠ object exists Prefect decides a hit from its own result record, not from COS. Delete the mp4 but keep the cache record ⇒ a hit returns a dangling key. Defend with an existence check (read_path/HEAD ⇒ re-encode if gone), or align cache_expiration with your COS lifecycle rule.
