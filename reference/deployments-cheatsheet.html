Deployments via serve()

Turn a script into a flow that runs itself on a schedule — local, no work pools. Copy-paste ready.

The one-liner

from prefect import flow

@flow(log_prints=True)
def pipeline(label: str = "nightly"):
    print(f"running {label}")

if __name__ == "__main__":
    pipeline.serve(name="my-deployment", cron="0 8 * * *")
What serve() is One call that (1) registers a deployment and (2) runs a long-lived process which creates and executes scheduled runs. It blocks — keep it running, Ctrl-C to stop. No workers, no containers.

serve() arguments

ArgumentWhat it does
name="..."Deployment name. Trigger it as <flow-name>/<name>.
cron="0 8 * * *"Clock-based schedule (standard cron syntax).
interval=3600Every N seconds. (Also accepts a timedelta.)
parameters={...}Default arguments passed to the flow on each scheduled run.
tags=[...]Labels for filtering runs in the UI / CLI.
paused=TrueRegister the deployment but don't schedule until resumed.

Schedules: cron vs interval

pipeline.serve(name="nightly", cron="0 8 * * *")    # every day at 08:00
pipeline.serve(name="hourly",  interval=3600)       # every hour
pipeline.serve(name="nightly", cron="0 8 * * *", parameters={"label": "prod"})

Trigger & inspect from the CLI

uv run prefect deployment ls
uv run prefect deployment run 'pipeline/my-deployment'   # one off-schedule run
uv run prefect flow-run ls --limit 5

The gotcha

No process, no runs serve() is the scheduler. Kill the process and the schedule stops firing. For real use keep it alive (launchd, tmux, systemd). The alternative that survives a logout — flow.deploy() + work pools + workers — needs infra that's out of scope while learning locally.
